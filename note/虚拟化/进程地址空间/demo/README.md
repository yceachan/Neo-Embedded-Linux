# 进程地址空间分析工具 (Address Space Analyzer)

本项目提供了一个用于动态分析 Linux 进程虚拟地址空间的命令行工具。通过该工具，你可以直观地看到一个运行中进程的内存分段情况（如代码段、数据段、堆栈、动态链接库等），并能够将该进程内存中**可执行段**的前 1KB 机器码实时反汇编为汇编指令。

## 1. 目录结构

- `hello.c`: 一个用于提供测试目标的简单 C 程序，其中特意分配了全局变量、静态变量和函数，以便在内存映射中形成清晰的 Data、BSS 和 Code 段。程序运行后将休眠 30 秒，留出充足的分析窗口。
- `as_analyzer.py`: 核心分析脚本（Python 3），负责解析 `/proc` 文件系统并调用系统底层工具进行反汇编。
- `README.md`: 本说明文档。

## 2. 原理架构

该工具的核心依赖于 Linux 内核提供的 `/proc` 伪文件系统以及 GNU Binutils 工具链。

### 2.1 内存映射解析 (`/proc/[pid]/maps`)
Linux 内核通过 `/proc/[pid]/maps` 暴露了进程的虚拟内存区域（VMA, Virtual Memory Areas）。`as_analyzer.py` 首先读取该文件，解析出进程内存空间的每一个连续块：
- **Address Range**: 虚拟地址的起始与结束范围（如 `55f5e3e2a000-55f5e3e2b000`）。
- **Perms**: 内存保护权限（`r` 读, `w` 写, `x` 执行, `p` 私有, `s` 共享）。
- **Offset**: 文件映射的偏移量（匿名映射通常为 0）。
- **Path/Region**: 映射来源，例如二进制执行文件路径、动态库文件路径、`[heap]`（堆）、`[stack]`（栈）或匿名映射 `[anon]`。

### 2.2 运行时内存读取 (`/proc/[pid]/mem`)
当脚本在 `maps` 中发现带有 `x`（可执行）权限的内存段时，说明这是代码指令所在的区域。此时脚本利用 `/proc/[pid]/mem` 伪文件，它可以当作整个进程的虚拟内存空间直接进行 `seek` 定位和读取。脚本定位到可执行段的首地址，读取出 1024 Bytes (1KB) 的原生机器码。
> **注意**：出于 Linux 安全机制（Yama ptrace 限制机制等），跨进程读取 `/proc/[pid]/mem` 通常需要 root 权限，这也是分析器需要 `sudo` 执行的原因。

### 2.3 动态反汇编 (GNU `objdump`)
脚本读取到机器码后，将其存入临时文件，然后调用 `objdump` 将其反汇编。由于这里提取的是纯机器码，我们使用了以下特殊参数：
- `-b binary`: 告知 objdump 输入文件没有标准 ELF 头，是纯二进制流。
- `-m <arch>`: 脚本通过 `platform.machine()` 自动识别宿主机架构（x86-64 或 arm64），告知 objdump 使用相应的指令集进行解码。
- `--adjust-vma=<addr>`: 由于是纯二进制，objdump 默认从地址 `0x0` 开始排布指令。使用此参数可以将反汇编显示的基址调整为进程真实的虚拟内存起始地址，便于指令地址的严谨对照。

## 3. 使用流程

### Step 1: 编译测试程序
在当前目录下，将 `hello.c` 编译为可执行文件：
```bash
gcc -o hello hello.c
```

### Step 2: 启动测试目标
在后台启动该程序，它会打印出自身的进程 PID 和各个变量的内存地址，随后进入 30 秒的休眠状态等待分析：
```bash
./hello &
```
*你可以记下输出日志中的 PID（例如 `PID: 12345`），或者利用 shell 变量 `$!` 获取后台进程 PID。*

### Step 3: 执行地址空间分析
使用 `sudo` 运行 Python 分析脚本，并传入刚才测试程序的 PID：
```bash
sudo ./as_analyzer.py <PID>
# 例如：sudo ./as_analyzer.py $!
```

### Step 4: 观察输出与清理
脚本会输出目标进程完整的地址空间映射表。当扫描到某个内存区域包含 `x` 权限时（如主程序的 `.text` 段，或者 `libc.so` 的可执行段），会自动在映射表下方附加上该区域首 1KB 的反汇编代码片段（受限于篇幅，最多打印前 30 行）。

分析完成后，可以手动杀掉测试进程（如果它尚未自动退出）：
```bash
kill <PID>
```
